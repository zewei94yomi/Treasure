# 宝物与图鉴设计

> 项目：网页版本地双人寻宝游戏（俯视 2D、视野/阴影系统、美术风格参考鹅鸭杀）
> 玩法定位：搜打撤 —— 进图搜宝箱 → 躲避/周旋怪物 → 带着宝物撤离才算真正到手

> **v3 更新（2026-07-12）**：宝物扩至 **72 件 / 11 系列**——新增 6 个地图专属系列（王庭旧梦/地心结晶/深海秘藏/晦暝圣物/瘴泽遗物/永冻奇物，各 6 件，只在对应地图掉落且在本图权重 ×1.6），堕天使之羽为地狱难度独占；收藏家等级阈值调整为 0/500/1400/2800/4500。
> 系统扩展：怪物 8 种（新增幽火/裂形泥怪/尖啸者，全员攻击前摇硬直）、武器 12 种（新增影袭匕首/寒霜喷罐/轰天雷）、药品 6 种、护甲 3 档+腰包、负重系统（重量→移速/脚步声/被发现距离）、潜行、神秘商人（困难+地狱，55% 收购宝物但**不计图鉴**）、地狱难度、6 张地图（血月教堂 怪物视野+10%、迷雾沼泽 玩家视野-18%、冰湖洞窟 冰面打滑）。

---

## 1. 设计原则

1. **撤离才算收录**：拾取宝物 ≠ 拥有。只有活着带到撤离点，宝物才计入图鉴和资产。被怪物抓住会掉落身上的宝物。这是搜打撤模式的核心张力。
2. **随机但有惊喜曲线**：宝箱位置和内容随机，但开箱概率、保底机制、难度独占宝物都经过设计，保证"每局都有盼头"。
3. **图鉴驱动重开率**：图鉴是长线目标。稀有度、难度独占、系列集齐奖励共同构成"再来一局"的理由。
4. **部分宝物有玩法效果**：少数宝物不只是分数，还与视野系统、怪物 AI 互动（加视野、显示怪物、甚至带诅咒），让"捡到什么"影响本局策略。

## 2. 稀有度体系

| 稀有度 | 颜色 | 基准价值（金币） | 体积（背包格） | 图鉴点数 | 定位 |
|---|---|---|---|---|---|
| 普通 Common | 灰白 | 20 – 50 | 1 | 10 | 保底收获，凑数也开心 |
| 稀有 Rare | 蓝 | 80 – 150 | 1 – 2 | 25 | 常规惊喜 |
| 史诗 Epic | 紫 | 200 – 400 | 1 – 3 | 60 | 单局亮点，部分带效果 |
| 传说 Legendary | 金 | 500 – 800 | 1 – 4 | 150 | 值得为它冒险 |
| 神话 Mythic | 红 | 1200 – 2000 | 3 – 6 | 400 | 每局全图至多 1 件 |

**负重规则**：携带传说宝物移速 −5%，神话 −10%（可叠加，上限 −15%）。大宝贝 = 大目标，强化"贪还是撤"的抉择。

**背包**：基础 8 格。体积让玩家在"塞满小件"和"扛一件大的"之间做取舍。

## 4. 宝箱设计（宝物的来源）

| 宝箱 | 外观 | 掉落分布 | 出现规则 |
|---|---|---|---|
| 木箱 | 朴素木质 | 普通 70% / 稀有 25% / 史诗 5% | 各难度大量分布 |
| 银箱 | 银边、微光 | 普通 40% / 稀有 40% / 史诗 17% / 传说 3% | 普通难度起，多在危险区 |
| 金箱 | 金光、粒子特效 | 稀有 35% / 史诗 45% / 传说 17% / 神话 3% | 全图 1–3 个，必在怪物巡逻核心区 |
| 神秘箱 | 轻微颤动、锁孔发红光 | 50% 是**宝箱怪**（伪装怪物，开启即袭击）；否则按金箱掉落且神话率 5% | 困难模式常驻，普通难度偶现 |

**难度联动**：
- 简单：木箱为主，无神秘箱，宝物不掉稀有以上时有安慰性金币。
- 普通：银箱增多，金箱 1–2 个。
- 困难：金箱 2–3 个 + 神秘箱常驻 + 2 件独占宝物（怪物的獠牙、世界树种子）。**想集齐图鉴必须挑战困难**。

**保底与防重复（对收集友好）**：
- 开箱 roll 宝物时，**未收录**的宝物权重 ×1.5。
- 连续 20 个宝箱未开出史诗及以上 → 下一箱必出史诗+（保底计数持久化存档）。
- 神话每局至多 1 件：全图生成时预先决定"本局是否有神话、藏在哪个金箱/神秘箱"。

**位置随机但精心设计**（简述，后续出关卡文档细化）：地图划分区域，每区域有一组手工布置的宝箱候选点位；生成时按约束抽选——高级箱远离出生点、朝怪物密度高的区域倾斜、任意两箱最小间距、撤离点附近不放金箱（避免白捡）。

## 5. 图鉴功能设计

### 5.1 收集状态（三态）

| 状态 | 展示 | 触发 |
|---|---|---|
| **未发现** | 黑色剪影 + `???`，仅显示稀有度底色 | 默认 |
| **已目击** | 灰度图 + 名称，详情锁定，标注"曾经拥有…" | 拾取过但未能成功撤离 |
| **已收录** | 全彩大图 + 全部详情 | 携带该宝物成功撤离 |

"已目击"是搜打撤的灵魂状态——你见过它、摸过它、却把它丢在了怪物爪下。图鉴里那格灰色就是下一局的动力。

### 5.2 图鉴主界面

- **入口**：主菜单「宝物图鉴」按钮 + 每局结算页的"查看图鉴"快捷入口。
- **布局**：按 6 个系列分组的网格（每系列一行 6 格，或系列页签切换）。整体风格与游戏一致：暗色羊皮纸/博物馆展柜质感，稀有度用边框光效区分。
- **每格显示**：宝物图标（按三态渲染）、稀有度边框、右上角 `NEW` 角标（收录后首次查看前显示）。
- **顶部总览**：总收集进度（`23 / 36`）+ 环形进度条 + 收藏家等级。
- **筛选/排序**：按稀有度、按系列、只看未收录、只看"已目击"。
- **系列行尾**：该系列进度（`4/6`）+ 集齐奖励图标（未领取时发光）。

### 5.3 宝物详情页

点击已收录的格子弹出：

- 大图 + 名称 + 稀有度 + 所属系列
- 风味文本（图鉴才能看到完整版）
- 效果说明（若有）
- **获取途径提示**：出自哪些宝箱 / 是否难度独占（对未收录宝物也显示此条，作为寻宝指引）
- 收藏数据：**首次收录时间、累计带出次数、单件累计产出金币**

### 5.4 收藏家等级与集齐奖励

- **收藏点数**：按稀有度给点（见稀有度表），只在**首次收录**时获得。
- **收藏家等级**：点数累计升级，如 见习寻宝人 → 寻宝行家 → 藏品鉴赏家 → 传奇收藏家 → 午夜馆长。等级解锁：玩家角色新配色、图鉴界面皮肤、称号（显示在结算页）。
- **系列集齐奖励**（以装饰为主，少量轻功能，避免破坏难度平衡）：

| 系列 | 集齐奖励 |
|---|---|
| 阁楼杂物 | 称号「破烂之王」+ 角色复古配色 |
| 海盗遗产 | 角色海盗帽装饰 |
| 失落文明 | 图鉴"石板"主题皮肤 |
| 珠光宝气 | 角色宝石光环特效 |
| 诡秘藏品 | 称号「不信邪」+ 幽灵宠物跟随（纯装饰） |
| 神话遗物 | 金色结算界面 + 称号「午夜馆长」 |
| **全图鉴** | 隐藏称号「传说本身」+ 主菜单彩蛋动画 |

### 5.5 结算流程中的图鉴时刻

撤离成功 → 结算页逐件翻出本局带出的宝物（卡牌翻面动画，稀有度越高特效越夸张）→ 新收录的宝物打上 **「新收录！」** 印章并飞入图鉴 → 显示收集进度变化（`22/36 → 24/36`）。失败被抓时，展示"本局失去的宝物"，未收录的标记为"已目击"。

### 5.6 双人模式的收录规则

本地同屏双人共用一份存档：**任意一人成功带出宝物，即计入共享图鉴**。合作寻宝、共同集邮——一人吸引怪物、一人扛宝撤离的配合本身就是玩法。结算页分别展示两人各自带出的部分以保留个人成就感。

### 5.7 数据结构与持久化

纯前端本地游戏，用 `localStorage` 持久化（键：`treasure-hunt.save.v1`），版本号字段留好迁移余地。

```ts
type Rarity = 'common' | 'rare' | 'epic' | 'legendary' | 'mythic';
type SeriesId = 'attic' | 'pirate' | 'civilization' | 'jewelry' | 'occult' | 'mythic_relics';
type Difficulty = 'easy' | 'normal' | 'hard';
type ChestTier = 'wood' | 'silver' | 'gold' | 'mystery';

/** 静态宝物定义（打包在代码里，共 36 条） */
interface TreasureDef {
  id: string;             // 如 'pirate_compass'
  name: string;
  series: SeriesId;
  rarity: Rarity;
  value: number;          // 金币
  size: number;           // 背包格
  flavor: string;         // 风味文本
  effect?: {
    kind: 'on_pickup' | 'while_carried' | 'active_once';
    description: string;
    // 具体参数由效果系统解析，如 { visionBonus: 0.15 }
    params: Record<string, number>;
  };
  acquisition: {
    chestTiers: ChestTier[];
    minDifficulty?: Difficulty;   // 难度独占
    unlockCondition?: 'all_others_collected';  // 「午夜」皇冠
  };
}

/** 图鉴存档（动态） */
type CodexState = 'unknown' | 'seen' | 'collected';

interface CodexEntry {
  state: CodexState;
  isNew?: boolean;            // NEW 角标，查看后清除
  firstCollectedAt?: string;  // ISO 时间
  totalExtracted: number;     // 累计带出次数
  totalGoldEarned: number;    // 单件累计产出
}

interface SaveData {
  version: 1;
  codex: Record<string, CodexEntry>;   // key = TreasureDef.id
  collectorPoints: number;
  claimedSeriesRewards: SeriesId[];
  pityCounter: number;                 // 史诗保底计数
  stats: { totalRuns: number; totalExtractions: number; totalGold: number };
}
```

收录判定的唯一入口是"撤离结算"函数：结算时遍历背包 → 更新 `codex`、点数、保底计数 → 写回 localStorage。游戏内任何其他地方不直接改图鉴数据，避免状态不一致。

## 6. 武器 / 弹药 / 耐久 / 商店（已实装）

### 6.2 弹药与补给

- 弹药三系：🔹轻型（30发/120金）、🔸霰弹（10发/100金）、🔺重型（10发/180金）；开箱有概率白捡。
- 补给：🩹医疗绷带（+55HP/100金）、🥤能量汽水（移速+30% 8秒/80金），团队共享，智能使用键。

### 6.3 商店（鸭可夫商店）

局外系统：买武器 / 买弹药 / 买补给 / 军械库维修。货币为撤离带出宝物换得的金币。军械库中每把武器是独立实例（各自耐久），双人各选一把带入局。
