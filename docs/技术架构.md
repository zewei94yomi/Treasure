# 技术架构

> 封存版总览（v31，2026-07）。下方历史小节按轮追加，本节是最终形态的一张图。

## 0. 总览

**技术选型**：纯 HTML5 Canvas + 原生 JS（非模块 script 标签按序加载共享全局），**零依赖、零构建、零服务端**。整站静态，游戏运行期无网络请求。

**启动链**：`main.js` DOMContentLoaded → 画布自适应缩放 + 输入监听（键盘全局、鼠标含 DOM 准星直驱）→ UI 菜单。`new Game(mode, diff, loadouts, mapId, skins, opts)` 构造即开局（构造器先销毁旧局 rAF），`opts` 旗标决定形态：`{horde, escape, arena, versus}`。

**帧循环**：`frame(t)` → `update(dt)`（输入→实体→模式引擎→武器掉落→天气→道具→相机→HUD DOM）→ `render()`（每玩家一个视口：地图预渲染画布 → 实体 → 特效 → 视野黑暗层(overlay canvas destination-out) → 天气 → HUD 画布元素）。双人纵向分屏，各自独立相机。

**关键系统索引（终版）**
- **视野**：`vision.js` 射线步进多边形挖洞 + 径向渐变；致盲=光照半径×0.32。
- **寻路**：怪物=多源 BFS 流场（玩家为源，0.4s 重算，8 邻梯度下降禁斜穿）+ 分离推挤 + 卡死侧滑；佣兵=视线受阻时走流场（跟主人）或 bfsPath（打怪）+ 侧滑；经典模式怪保留巡逻/调查/追击状态机。
- **割草升级**：`hordeState.P[idx]` 每玩家独立 {level, xp, skills(20), mods, picked, exT}；世界物件（地雷/陨石/锯盘/鸭灵/手雷/轰炸机）共享在 `H.ex` 供绘制层遍历。
- **调参三层**：TUNE_DEFS(全局) / HERO_TUNE / MONSTER_TUNE / SKILL_TUNE，四个 `xxxVal(id,key)` 读取器统一「存档覆盖 > 代码默认」；全部游戏公式接读取器，图鉴/卡面文案同源。
- **DDA**：3s 采样血线偏离 0.6 基准 ±死区，满血无伤加压，幅度受 `ddaStr` 钳制，作用于刷怪间隔。
- **对决(versus)**：独立旗标（非 horde）：互伤管线（子弹/近战/爆炸对玩家判定）、回合三胜、读秒冻结、空投、临时武器复制。
- **音频**：WebAudio 合成器（tone/noise 包络）+ 采样层（Audio cloneNode 并发，CC0 素材，失败回退合成）。
- **准星**：DOM 元素 mousemove 直驱（零画布帧延迟），Kenney 素材染色 data-URI。

## 工程备忘（血泪坑）
1. `?v=N` 全站一致 + index.html 自身也会被启发式缓存（开发用 no-store 服务器）。
2. 后台标签 rAF 冻结 → E2E 用 `g.frame(t)` 手动泵帧，时间基准字段是 `g.last`。
3. python patch() 锚点必须唯一且带上下文——`levelupQueueCheck` 在 update() 与 closeMerchant() 各有一处，曾错插。
4. CSS 入场动画的结束态必须与静态 transform 一致，否则动画结束瞬间跳变（升级卡闪缩案）。
5. 共享可变状态是重构最大成本源：升级分离改了 60+ 处引用；新系统尽量一开始就按 owner 归属建模。
6. 面板滑杆浮点比较用 epsilon；`u.mod(mods)` 是不可逆乘算，调试台"降级"只能整体重置。


纯 HTML5 Canvas + 原生 JS，零依赖零构建。多 `<script>` 经典加载（非 ESM），file:// 可直接运行。

## 模块分层与加载顺序

```
sfx.js / music.js     音频层（WebAudio 合成音效 + 程序化 BGM：menu/game/battle 三主题）
skins-data.js         皮肤贴图 data-URI（Kenney CC0，构建自 assets/skins/）
data.js               纯数据：宝物/武器/弹药/药品/护甲/怪物/难度/皮肤/天气/道具/割草配置
save.js               localStorage 存档、图鉴/奖杯/军械库读写（购买即落盘 + 读回校验）
map.js                6 张 ASCII 地图注册表、parse、碰撞/视线/BFS、宝箱布点、小地图底图
vision.js             射线步进视野多边形 + 黑暗层合成
entities.js           Player / Monster(10种AI) / Bullet / Mercenary / Chest / XPGem 等实体
game.js               Game 类：主循环、战斗、开箱、撤离、分屏渲染、HUD、结算
─── 以下为 Game.prototype 扩展模块（新功能不再膨胀 game.js）───
horde.js              无双割草扩展：智能三选一、6 个新技能引擎、局内商人
weather.js            天气系统：随机全局效果 + 粒子/色调渲染 + 倍率查询
powerups.js           地图随机道具系统：7 种道具生成/拾取/生效
ui.js                 全部 DOM 界面（菜单/配装/商店/图鉴/奖杯/商人/升级三选一/结算）
trainer.js            键位练习场：设置页内的双人迷你沙盘（独立 rAF，读取实时 KEYMAP）
main.js               引导：读档、皮肤预载、音频解锁、beforeunload 落盘
```

**解耦模式**：横切功能用 `Object.assign(Game.prototype, {...})` 从独立文件挂载，
零调用点改动、物理分文件；`horde.js` 中的 `openLevelup` 覆盖 game.js 基础版（加载顺序即优先级）。

## 大逃亡与地图放大（第九轮）
- `scaleAscii()`（map.js）：加载时每格→2×2；唯一点位(1/7/S)只留左上，密度点位(c/g/m/t)留对角两份，结构字符铺满——`MapData.ascii` 携带放大后网格（prerenderMap 必须读它而非 def.ascii）
- `genEscapeAscii()`：房间链生成器，先排房再定宽、L 形走廊构造即连通；`MAPS.escape.gen` 标记每局重生成，`noScale` 跳过二次放大
- 大逃亡 = `opts.escape`（this.horde 同置 true 复用割草体系）：`escapeUpdate` 潮汐/追兵/商人、`escapeLoot` 开箱掉枪、`settleEscape` 里程结算；hordeUpdate 用 else 包住割草专属段
- 伤害数字统一从 `Monster.hurt` 冒出（entities.js），渲染在 floaters 之前；玩家受伤红色由 damagePlayer 冒
- 皮肤描边：preloadSkins onload 烘焙八向黑剪影 canvas 替换 SkinImages（附 drawScale 补偿内边距）

## UI 设计体系（第十九轮 · 方向A 野战档案）
- `css/style.css`：全量设计令牌（`--amber/--ink/--line/--p1/--p2` 等）+ 全组件军规化；选择器契约与 JS 生成标记保持兼容（改样式不改类名）。
- `css/fonts.css` + `assets/fonts/*.woff2`：四字体自托管（ZCOOL QingKe HuangYou / Black Ops One / Share Tech Mono / Noto Sans SC），unicode-range 子集按需加载，无外部 CDN 依赖。
- 主菜单简报档案由 `ui.js showMenu()` 注入 `#menu-info`（金币/头衔/图鉴奖杯进度/随机情报）。
- HUD 血条动态色在 `game.js` 中按血量百分比切换军规渐变（绿 #9cbb61 → 黄 #d9a441 → 红 #c2452a）。
- 设计稿原件：用户 Downloads『惊魂寻宝鸭UI设计-离线版.html』（打包格式：manifest+template JSON，node 可解包）。

## 第二十轮技术要点
- **铭牌 HUD**：index.html 三层 DOM（#hud-task / #hud-world / .nameplate×2）；game.js `updateHudLayers()` 按模式切换任务层文案，世界层 top/right 按 `canvas.clientWidth / canvas.width` 显示缩放换算贴住小地图；击杀播报 `this.feed[{txt,until}]`（割草只报精英/Boss）；神话警报 `mythicAlertUntil`。铭牌更新走 updateHud 既有循环（_sig 签名防抖）。立绘 = 每 30 帧离屏调 drawPlayer（注意 p.active 是 getter 不可写）。
- **自定义下拉 `enhanceSelect(sel, {icon, tag})`**（ui.js）：隐藏原生 select 保状态，div 重建按钮+列表，选项 value 为空视为分组标签；选中派发 change 事件与原 onchange 兼容。
- **怪物档案照**：浏览器离屏用 `Object.create(Game.prototype)` mock（需给 players 数组，boss 用 `{...type, boss:false}` 副本避免头顶血条入画）跑 drawMonster → 亮度伽马 0.72 → 6 阶骨白版画分色 → 帆布/颗粒/角括号/SPEC 编号/盖章 → webp(0.88) 160px。数据文件 js/mugshots-data.js（121KB/21 张），独立素材 assets/mugshots/*.webp。提取路径：页面 fetch POST 到本地 node 接收器（工具直传会被安全过滤拦截）。
- **调参默认值固化**：TUNE_DEFS 与 HERO_TUNE 的 def 直接改为用户配置；欲望公式 `hv('desire',380) × tune('mercDesire')/380`，全局与每英雄双轨不打架。


## 第二十一轮架构变更
- **升级分离**：`hordeState.P[idx] = { level, xp, xpNeed, freeChoices, skills, mods, picked, missileT/novaT/boltT/trailT, exT }`；`Game.hsP(p)` 取玩家子状态；世界物件（mines/meteors/booms/pets/whirlFx）仍共享在 `hordeState.ex` 供绘制层遍历，冷却计时在各自 `exT`。佣兵相关加成一律走 `hsP(mc.owner)`。
- **对决引擎**：`opts.versus` → `this.vs = { score, round, target:3, freezeT, roundEndT, matchOver }`；`versusUpdate/versusKill/resetVersusRound/settleVersus`；互伤在 Bullet.update（玩家碰撞）、tryAttack 近战弧、explode 波及（本人 0.5 倍）；冻结用 freezeT/roundEndT 同时闸住移动/射击/翻滚。
- **调参三层**：`HERO_TUNE/heroVal`、`MONSTER_TUNE/monsterVal`（构造血量、行动速度、resolveAttack 伤害、Boss/远程/突进参数）、`SKILL_TUNE/skillVal`（19 技能全公式接线，dex.js SKILL_INFO 同步读取）——UI 共用 `renderTuneTable/setTuneVal` 通用滑杆表。
- **Boss 技能**：Monster 侧 `bossSkillT` 计时 → `pendingBossSkill` 前摇 → `Game.bossSkill(m)` 结算；致盲 `blindPlayer(p,dur)` → 视野光照半径 ×0.32。
- **潜行**：`MapData.bushes`（生成于 gen 装配处）、`bushHides(p)` 挂进 `Monster.canSee` 与潜伏者觉醒；投石 `throwPebble` → `pebbles` 飞行 → `emitNoise(x,y,300)`。
- **坑**：`levelupQueueCheck` 在 update() 与 closeMerchant() 各有一处调用——patch 锚点务必带上下文，本轮曾把主循环调用误插进 closeMerchant。


## 第二十二轮架构变更
- **流场寻路**：`hordeFlowUpdate`（多源 BFS，Int16Array，0.4s 节流，240 格截断）+ `flowDir`（8 邻梯度下降、禁斜穿墙角）；`moveToward` 中割草追玩家时优先流场并跳过单怪 BFS。
- **DOM 准星**：`#aim-cursor` 由 main.js mousemove 直接 transform（position:fixed），画布准星移除；`SAVE.settings.hideCursor` 控制 canvas cursor:none。
- **佣兵复活**：mercs 过滤改为保留 `reviveT !== undefined` 的尸体，倒计时归队；`noRevive` 标记主人阵亡带走的英雄。
- **基础强化**：HORDE_UPGRADES 被动卡的 mod lambda 读 `skillVal('base', …)`，`descFn` 动态文案（升级卡与技能图鉴同源）。
