# 故障排查（Troubleshooting）

> 按"症状 → 原因 → 处理"整理。开发期真实踩过的坑都在这里。

## 玩家侧

**改完代码/更新后没变化、或界面错乱**
浏览器缓存了旧脚本。线上：硬刷新（⌘⇧R / Ctrl+F5）。开发：确认 `?v=N` 已 +1；本地调试用 no-store 服务器（见《运行与配置指南》§1）。`index.html` 本身也会被缓存——给地址加个随机查询串。

**存档"丢了"**
九成是**源不一样**：github.io、`localhost:8787`、`localhost:8011`、`file://` 各有一份独立 localStorage。回到原来的地址即可。真正会清档的只有：浏览器清站点数据、隐私模式、`file://` 下部分浏览器拒写（主菜单有红色警告条提示）。

**按键没反应 / 键位怪**
① 中文输入法接管了 `/ 。 ，` 等键——切英文输入法（游戏有全角回退映射，但英文态最稳）。② 双人同按多键触发键盘鬼影（硬件矩阵限制）——错开键区或改绑（键位设置内左下有"最近按键"自查区）。③ 改绑冲突时游戏会自动互换键位，看提示。

**没有声音**
浏览器自动播放策略：首次交互（点击/按键）后音频才解锁。音乐可在设置中心单独开关；采样音效（翻滚/冰冻）加载失败会自动回退合成音，不会静音。

**准星不见了 / 鼠标不见了**
准星只在**单人+鼠标操控开**时显示；系统鼠标默认隐藏（设置中心「隐藏系统鼠标只留准星」可关）。双人模式无准星（纯键盘）。

**卡顿**
全局调参把「同屏怪物上限」（默认 80）往下调；关闭「打击特效」；割草后期上千实体属正常范围，Chrome 表现最好。

**难度不对劲（太简单/太难）**
先查全局调参是否有历史覆盖（行高亮=改过，「恢复默认」一键回厂）；再调「自适应强度」（0=关，默认 0.3，越大波动越猛）。注意 v31 默认已含"全怪+10%、Boss+15%"平衡。

**割草里佣兵/技能表现和文案对不上**
技能图鉴、升级卡文案实时读调参面板——检查 ✨ 技能调参 / ⚔️ 英雄调参是否有覆盖值。

## 开发侧

**改了 data.js 默认值但游戏里没生效**
存档里的调参覆盖优先于代码默认。清掉存档对应键（或面板「恢复默认」）再验。

**测试挂了一片**
`tests/run-tests.js` 的断言是"当前态快照"：改常量/数量（英雄数、卡池数、TUNE_DEFS 项数、公式字符串）必须同步更新对应断言——这是设计行为，逼你确认改动影响面。

**python 补丁脚本报"锚点不存在/不唯一"**
patch() 断言失败即整文件不写入（同批已通过的 pair 也不落盘），修正锚点后**整批重跑**。锚点务必带足上下文——曾把主循环调用误插进 `closeMerchant()`（两处同名行）。

**新建对局后旧局还在跑 / 重置无效**
必须走 `new Game(...)` 构造（内部会销毁旧局的 rAF 循环）。手动测试时若直接改状态，记得 `Game.current.over = true; cancelAnimationFrame(Game.current.raf)`。

**浏览器 E2E 里 rAF 不走帧**
后台标签页 rAF 冻结。用泵帧法：`let t = g.last; for(...){ t += 16.7; g.frame(t); }`——注意字段是 `g.last`（传 NaN 会毒化所有计时器）。

**佣兵/怪物卡地形**
怪物走多源 BFS 流场（0.4s 重算，`hordeFlowUpdate/flowDir`）；佣兵视线被挡走流场（跟主人）或 bfsPath（打怪），卡死 0.3s 侧滑。若再出现：检查地图 `solid` 网格与 `resolveCircle` 半径是否匹配。

**回滚一轮改动**
每轮一个 commit，`git log --oneline` 找到轮次 → `git revert <sha>`。存档层面的回滚见 `backups/` 目录说明。
