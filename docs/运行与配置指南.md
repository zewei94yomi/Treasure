# 运行与配置指南（Setting Up / 启动 / 配置）

> 封存版（v31）。本文覆盖：怎么跑起来、存档在哪、每个配置入口是干什么的、怎么发版。

## 1. 三种启动方式

| 方式 | 命令/地址 | 适用 | 注意 |
|---|---|---|---|
| **线上（推荐）** | https://zewei94yomi.github.io/Treasure/ | 直接玩 | 存档在浏览器 localStorage，换浏览器不互通 |
| **本地服务器** | `python3 -m http.server 8787` → `http://localhost:8787` | 本地玩/改代码 | 与线上是**不同源**，存档独立 |
| **直接双击 index.html** | `file://` 协议 | 应急 | 部分浏览器禁写 localStorage，主菜单会出红色警告条；不推荐 |

**开发迭代**（改 JS 即时生效，绕过缓存）用 no-store 服务器：

```bash
python3 -c "
from http.server import SimpleHTTPRequestHandler, HTTPServer
class H(SimpleHTTPRequestHandler):
    def end_headers(self):
        self.send_header('Cache-Control', 'no-store')
        super().end_headers()
HTTPServer(('127.0.0.1', 8011), H).serve_forever()"
```

## 2. 缓存约定（改代码必读）

- 所有 js/css 以 `?v=N` 引用（当前 **v=31**），全站同一个 N，由 `tests/run-tests.js` 断言一致性。
- **发版必须把 index.html 里所有 `?v=N` +1**，否则玩家拿到旧脚本。
- `index.html` 本身也会被浏览器启发式缓存：本地验证时给页面地址加随机查询串（如 `?dev1`），或用上面的 no-store 服务器。
- 音频/图片资源按文件名新增，不参与 `?v=` 计数。

## 3. 存档（localStorage）

- **键名**：`treasure-hunt.save.v1`；**按源隔离**（github.io / localhost:8787 / localhost:8011 是三份独立存档）。
- **结构**（JSON 单键）：`gold`、`codex`（宝物图鉴逐件状态）、`weapons/armors/ammo/consumables/pouches`（军械库）、`mercTrials`（佣兵试用券）、`trophies`、`settings`（音乐/鼠标操控/隐藏鼠标/皮肤/devMode/上次配装）、`stats`（含 `mKills` 怪物讨伐数）、`monsterSeen`、`hordeBest/escapeBest`、以及四套调参覆盖 `tuning / heroTuning / monsterTuning / skillTuning`（面板改动才会出现；等于默认值时建议清掉）。
- **备份**：控制台 `copy(localStorage.getItem('treasure-hunt.save.v1'))` 即复制全文；**恢复**：`localStorage.setItem('treasure-hunt.save.v1', <粘贴>)` 后刷新。
- **跨机迁移**：把上述 JSON 拷过去即可，无服务器依赖。
- `backups/` 目录（不入库）存放历史清除的调参备份，如 `r23-cleared-tuning-backup.json`。

## 4. 配置入口一览（游戏内）

**设置中心**（主菜单 06）：键位设置（含双人练习场实时试键）、🎛️ 策划调参、👹 怪物调参、✨ 技能调参、🖱️ 鼠标操控开关、🎯 隐藏系统鼠标只留准星、🎵 音乐、🧪 开发者模式、🎯 调试练习场。

**四大调参面板**（实时生效、写入存档、行高亮=改过、恢复默认一键回厂）：

| 面板 | 数据源 | 管什么 |
|---|---|---|
| 🎛️ 全局（28 项） | `TUNE_DEFS`/`tune(id)` | 玩家/怪物全局倍率、DDA、天气时长与强度、大逃亡怪量、潮速、同屏上限、割草时长、开发者经验 |
| ⚔️ 英雄（12 人） | `HERO_TUNE`/`heroVal(hid,key)` | 每位佣兵的生命/伤害/攻速/射程/弹夹/换弹/移速/攻击欲望 + 专属技能参数 |
| 👹 怪物（21 种） | `MONSTER_TUNE`/`monsterVal(mid,key)` | 每种怪血/伤/速倍率、平A间隔 + 全部技能参数（Boss 技能细项在各 Boss 名下） |
| ✨ 技能（21 组） | `SKILL_TUNE`/`skillVal(sid,key)` | 割草 20 技能的基础值与每级增量 + 「基础强化」16 项每张卡增量 |

> 调参优先级：存档覆盖 > 代码默认（def）。第 20/23 轮曾把实测手感**固化为代码默认**——想复刻这个流程：面板调好 → 把存档里的覆盖值写进 data.js 各 `def` → 清掉存档覆盖。

**练习场工具条**：选武器试用 / 选英雄入队 / 放置怪物（21 种+一波+精英）/ 免费升级 / 重置木桩 / 全部重置 / 🎓 技能学习台（技能升降级 + 佣兵直招 + 49 张数值变体卡）+ 四面板直达。

**开发者模式**：金币恒 999999、全皮肤装饰解锁、经验倍率 `devXp`（默认 ×20）。

## 7. 部署形态

GitHub Pages「从分支部署」（main 根目录），无构建、无 workflow、无服务端。整站为静态文件，游戏无任何网络请求（字体/贴图/音频全部自托管或内联 data-URI）。
